When do you use Arrays vs Hashes?

October 25, 2015

I like to think of hashes as a refined array. Arrays are a data structure made up of units, each unit holding some value. The units are referenced by some default numerical index, starting from 0. Arrays default to one dimension, but can be stacked to create multiple dimension arrays. '''ruby my_array = [["Apple",1.50],["Banana",1.00],["Orange",1.00]] '''

Hashes are also a data structure, however, a hash has both a key and a value. The key replaces the index as a defined object or property of the value. An example of this is a dictionary: the key would be the word, and the value would be the definition.

'''ruby my_hash = {:Apple => 1.50, :Banana => 1.00, :Orange => 1.00 } '''

Hashes are much more efficient data structures. While arrays can store the same data, if you do not already know the index of an array, you will have to search the array to find the value. With a hash, you know the key, so you simply return the value of that key. However, if you need to iterate over a series of data, arrays are likely the way to go, as Ruby methods make it easy to iterate over the array. '''ruby #Calling the price for an Apple from an array #Note that you must know the index of the Apple before calling it p my_array[0,1] ''' '''ruby #Calling the price of an Apple from a hash p my_hash[:Apple] '''
