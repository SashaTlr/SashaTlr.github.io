The group_by Enumerable Method

October 29, 2015

What is the Ruby group_by method and how does it work?

Inputs: A collection

Outputs: A hash

The groups_by method will take a collection such as an array as an input, and transform it into a hash, which is an array with defined indices or keys.

The process that transforms the collection input into a hash is contained in a block. The process is some sort of method, but if a method has not been assigned, it will default to a one-entry hash where the key is nil, and the values are the original collection.


              [1,2,3,4,5].group_by{}
              => {nil=>[1, 2, 3, 4, 5]}

When the block contains a method or process, the keys will be the result of the process instead of nil, and the value for each key will be an array. The array will contain the elements from the input collection that corresponds to the key. If multiple elements from the input collection have the same result, there will be multiple values in the hash for that result key.

In this example, we want to calculate the frequencies of each integer in my_array using the group_by method. 


              my_array = [1,2,2,3,4,4,4,5,6,6]
              my_array.group_by{|i| my_array.count(i)}
              => {1=>[1, 3, 5], 2=>[2, 2, 6, 6], 3=>[4, 4, 4]}

The output shows the frequency of each integer from the array as the key values of the hash, and the array associated with each key lists each integer that corresponds to that frequency.


              => {1=>[1, 3, 5], 2=>[2, 2, 6, 6], 3=>[4, 4, 4]}

group_by is a powerful method that can be used for filtering data and understanding patterns.
